What is Commit Signing?

Commit signing is also known as code signing. It uses cryptography (e.g., GPG, SSH, or S/MIME signatures) to add digital signature to transferred data. Receivers of the data (such as GitHub) can then verify the authenticity of the data. Verified commits will have a green verified tag as shown below.

All commits made using the GitHub interface will automatically be signed. Signing of local commits should be done through the Git client. For details, please see GitHub documentation on signing commits: https://docs.github.com/en/authentication/managing-commit-signature-verification/signing-commits

For this survey, we will refer to signed commits as Verified commits.
